# Web OS 포트폴리오 앱 테스트 프롬프트

아래 프롬프트를 브라우저 조작이 가능한 AI(Antigravity, Claude Computer Use 등)에게 그대로 전달하세요.

---

## 프롬프트 시작

당신은 웹 애플리케이션 QA 테스터입니다. 브라우저에서 `http://localhost:5173` 에 접속하여 "Web OS" 스타일의 포트폴리오 앱을 체계적으로 테스트하고, 발견된 모든 문제를 정해진 형식으로 보고해야 합니다.

### 이 앱이 뭔가?

브라우저 안에서 데스크톱 운영체제처럼 동작하는 웹 앱입니다.
- 화면 왼쪽에 앱 아이콘들이 세로로 나열되어 있습니다
- 아이콘을 **더블클릭**하면 드래그/리사이즈 가능한 윈도우가 열립니다
- 하단에 작업표시줄(시계, 열린 앱 목록)이 있습니다
- 총 12개 앱이 등록되어 있어야 합니다

### 앱 목록 (12개)

| 순서 | 아이콘 | ID | 이름 | 설명 |
|------|--------|-----|------|------|
| 1 | ⌨ | terminal | Terminal | 터미널 (백엔드 없으면 연결 실패 표시 정상) |
| 2 | ◈ | vscode | VS Code | VS Code 임베드 (백엔드 없으면 설정 가이드 표시 정상) |
| 3 | ✦ | claude | Claude | AI 채팅 (백엔드 없으면 API 키 미설정 안내 정상) |
| 4 | 📊 | monitor | 시스템 모니터 | GPU/CPU 모니터 (백엔드 없으면 로딩 표시 정상) |
| 5 | 🖥 | kasm | 원격 데스크톱 | KasmVNC (백엔드 없으면 설정 가이드 표시 정상) |
| 6 | ⚙ | settings | 설정 | API 키/URL 설정 (백엔드 없으면 빈 입력 폼만 표시 정상) |
| 7 | 👤 | profile | 프로필 | 자기소개, 스킬 바, 링크 |
| 8 | 📄 | resume | 이력서 | 학력, 교육, 프로젝트, 기술 스택 |
| 9 | 💼 | projects | 프로젝트 | 카드 그리드 + 필터 + 상세 보기 |
| 10 | 📝 | blog | 블로그 | 포스트 목록/리더 + 가이드 탭 |
| 11 | 📅 | timeline | 타임라인 | 세로 타임라인 (경력/프로젝트 연표) |
| 12 | ✉ | contact | 연락처 | 이메일/링크 + 메시지 폼 |

**중요**: 1~6번 앱은 백엔드(FastAPI)가 필요한 기존 앱입니다. 현재 백엔드가 실행되지 않고 있으므로, 이 앱들은 "연결 실패", "설정 가이드", "로딩 중" 등의 메시지가 표시되는 것이 **정상**입니다. 이것은 버그가 아닙니다. 7~12번 앱이 이번 테스트의 핵심 대상입니다.

---

### 테스트 절차

아래 순서대로 **모든 항목**을 테스트하세요. 각 항목마다 결과를 기록하세요.

#### STEP 1: 페이지 로드 확인

1. `http://localhost:5173`에 접속합니다.
2. 화면이 정상 로드되는지 확인합니다.
3. **확인할 것**:
   - 어두운 배경의 데스크톱이 표시되는가?
   - 브라우저 콘솔(F12 → Console)에 빨간색 에러가 있는가? 있으면 에러 메시지를 전문 복사
   - 화면 왼쪽에 앱 아이콘이 보이는가?
   - 아이콘이 총 몇 개인가? (기대값: 12개)
   - 아이콘이 화면에 다 보이는가? 아니면 잘려서 스크롤해야 하는가?
   - 하단에 작업표시줄이 보이는가?
   - 작업표시줄 오른쪽에 시계가 표시되는가?

#### STEP 2: 윈도우 매니저 기본 동작

아무 아이콘이나 하나 더블클릭하여 윈도우를 엽니다. (7번 프로필 추천)

4. **더블클릭**으로 윈도우가 열리는가?
5. 윈도우 타이틀바(상단 바)를 드래그하여 이동할 수 있는가?
6. 윈도우 가장자리를 드래그하여 크기를 변경할 수 있는가?
7. 타이틀바 왼쪽의 **빨간 동그라미 버튼** 클릭 → 윈도우가 닫히는가?
8. 다시 열고 **노란 동그라미 버튼** 클릭 → 윈도우가 숨겨지는가 (최소화)?
9. 최소화된 후 **작업표시줄**에서 해당 앱 탭 클릭 → 윈도우가 다시 나타나는가?
10. **초록 동그라미 버튼** 클릭 → 윈도우가 전체 화면(최대화)이 되는가?
11. 다시 초록 버튼 클릭 → 원래 크기로 돌아오는가?
12. 윈도우 2~3개를 동시에 열고, 하나를 클릭하면 맨 앞으로 오는가 (z-index 포커스)?

#### STEP 3: 프로필 앱 (👤 Profile)

프로필 아이콘(👤)을 더블클릭합니다.

13. 윈도우 안에 프로필 정보가 표시되는가?
14. **이름**: "김건희" + "Kim Gunhui"가 보이는가?
15. **직함**: "풀스택 개발자 / AI 엔지니어"가 보이는가?
16. **이메일**: "k1212gh@tukorea.ac.kr"이 보이는가?
17. **아바타**: 왼쪽에 동그란 원 안에 "김" 글자가 보이는가?
18. **링크 버튼**: "GitHub", "Email" 버튼이 보이는가? 클릭하면 새 탭으로 이동하는가?
19. **인용문**: "자연어로 소프트웨어를 설계하고..." 텍스트가 파란 왼쪽 선과 함께 보이는가?
20. **핵심 역량**: 스킬 바가 8개 보이는가? (Claude Code, Next.js, Supabase, Git, TypeScript, Tailwind, Python, Docker)
21. 각 스킬 바에 퍼센티지(90%, 85%, 80% 등)가 표시되는가?
22. 스킬 바가 **색상**이 채워져 있는가? (빨간, 파란, 초록 등)
23. 하단에 "📄 이력서", "💼 프로젝트", "📝 블로그" 3개 버튼이 보이는가?
24. **"📄 이력서" 버튼 클릭** → 이력서 앱 윈도우가 **새로 열리는가**?
25. **"💼 프로젝트" 버튼 클릭** → 프로젝트 앱 윈도우가 **새로 열리는가**?
26. 전체 내용이 윈도우 안에서 스크롤 가능한가? (내용이 잘리지 않는가?)

#### STEP 4: 이력서 앱 (📄 Resume)

이력서 아이콘(📄)을 더블클릭합니다.

27. 윈도우 상단에 "🖨️ 인쇄" 버튼이 있는가?
28. **학력 섹션**: "🎓 한국공학대학교 / 컴퓨터공학과 / 2019.03 ~"가 보이는가?
29. **교육 섹션**: "🏫 삼성 청년 SW 아카데미 (SSAFY) / 14기 서울 캠퍼스"가 보이는가?
30. **프로젝트 경험 섹션**: 5개 프로젝트 카드가 보이는가?
    - Synaptic (🟡 진행중)
    - ALOE MCP Copilot (✅ 완료)
    - 딥페이크 탐지 (✅ 완료)
    - 관통 프로젝트 (✅ 완료)
    - AgentOS (🟡 진행중)
31. 각 카드에 기술 스택 태그(파란 뱃지)가 보이는가?
32. **기술 스택 섹션**: Frontend, Backend, AI/ML, DevOps, Tools 5개 그룹이 그리드로 보이는가?
33. 전체 내용이 스크롤 가능한가?

#### STEP 5: 프로젝트 앱 (💼 Projects)

프로젝트 아이콘(💼)을 더블클릭합니다.

34. 상단에 "프로젝트" 제목과 필터 버튼들 (전체/진행중/완료/Web/AI·ML)이 보이는가?
35. 프로젝트 카드가 **그리드**(2열 이상)로 표시되는가? 아니면 세로로 1열인가?
36. 카드 5개가 보이는가? (Synaptic, ALOE MCP, 딥페이크, 관통 PJ, AgentOS)
37. 각 카드에 이모지, 제목, 설명, 상태(진행중/완료), 기술 태그가 보이는가?
38. Featured 프로젝트(Synaptic, AgentOS)에 "⭐ Featured" 표시가 있는가?
39. **필터 테스트**: "진행중" 클릭 → 2개만 보이는가? (Synaptic, AgentOS)
40. **필터 테스트**: "완료" 클릭 → 3개만 보이는가?
41. **필터 테스트**: "전체" 클릭 → 5개 다시 보이는가?
42. **카드 클릭** → 상세 보기로 전환되는가?
43. 상세 보기에서:
    - 큰 이모지가 보이는가?
    - 제목, 설명, 상태, 기간, 역할이 보이는가?
    - 기술 스택 태그들이 나열되는가?
    - "문제 정의", "핵심 기능", "주요 성과" 섹션이 보이는가?
44. **"← 목록으로" 버튼** 클릭 → 카드 목록으로 돌아가는가?

#### STEP 6: 블로그 앱 (📝 Blog)

블로그 아이콘(📝)을 더블클릭합니다.

45. 상단에 "📝 포스트" | "📖 가이드" 2개 탭이 보이는가?
46. **포스트 탭** (기본):
    - 왼쪽에 사이드바(카테고리 필터 + 포스트 목록)가 있는가?
    - 카테고리: All, TIL, Project Log, Tutorial 4개 버튼이 보이는가?
    - 포스트 1개 ("AgentOS 포트폴리오 프로젝트 시작")이 목록에 있는가?
    - 오른쪽에 포스트 본문이 렌더링되는가?
    - 본문에 "Project Log" 뱃지(초록)와 "🤖 자동 생성" 표시가 보이는가?
    - 본문 제목, 날짜, 태그가 보이는가?
    - HTML 콘텐츠(h3, p, ol, pre/code 블록)가 **올바르게 렌더링**되는가?
    - 아니면 HTML 태그가 텍스트로 그대로 보이는가? (예: `<h3>오늘 한 일</h3>`처럼)
47. **가이드 탭** 클릭:
    - Claude Code 셋업 가이드 내용이 보이는가?
    - 코드 블록(pre/code)이 어두운 배경으로 보이는가?

#### STEP 7: 타임라인 앱 (📅 Timeline)

타임라인 아이콘(📅)을 더블클릭합니다.

48. 세로 중앙선이 보이는가?
49. 선 위에 동그란 아이콘(💻, 🔍, 🤖, 🔒, 📡, 🏫, 🎓)이 보이는가?
50. 각 이벤트 카드(날짜 + 제목 + 설명)가 좌우 교차로 배치되는가?
51. **아니면** 모든 카드가 한쪽으로만 쏠려있는가?
52. "진행중" 표시가 있는 항목(💻, 🔍)의 동그라미에 펄스(깜빡이는 그림자) 애니메이션이 있는가?
53. 카드가 윈도우 밖으로 잘리지 않는가? (overflow 문제)
54. 윈도우를 작게 리사이즈했을 때 타임라인이 깨지는가?

#### STEP 8: 연락처 앱 (✉ Contact)

연락처 아이콘(✉)을 더블클릭합니다.

55. **연락 방법 섹션**: 이메일(📧)과 링크(🔗) 항목들이 보이는가?
56. 이메일 항목 클릭 → mailto 링크가 동작하는가?
57. **메시지 보내기 섹션**: 이름 입력 필드와 메시지 텍스트 영역이 보이는가?
58. 입력 필드에 텍스트를 **입력할 수 있는가**? (커서가 들어가고 글자가 보이는가?)
59. 입력 필드의 텍스트 색상이 **보이는가**? (배경과 글자색이 같아서 안 보이는 건 아닌가?)
60. "📨 전송" 버튼이 보이는가?

#### STEP 9: 작업표시줄 동작

여러 앱을 동시에 열어봅니다 (프로필, 프로젝트, 블로그 등 3~4개).

61. 하단 작업표시줄에 열린 앱들의 탭이 표시되는가?
62. 탭에 아이콘과 제목이 보이는가?
63. 현재 포커스된 앱의 탭이 **파란색으로 하이라이트** 되는가?
64. 탭 클릭 → 해당 윈도우가 맨 앞으로 오는가?
65. 이미 포커스된 앱의 탭을 다시 클릭 → 최소화되는가?
66. 앱 탭이 너무 많아서 작업표시줄이 **넘치는가**? 넘치면 어떻게 보이는가?

#### STEP 10: 콘솔 에러 최종 확인

67. F12를 눌러 개발자 도구 → Console 탭을 엽니다.
68. **빨간색 에러 메시지**가 있는가? 있다면 모두 복사합니다.
    - 단, `/api/` 로 시작하는 fetch 실패(404, network error)는 백엔드 미실행이므로 무시합니다.
    - `favicon.svg 404`도 무시합니다.
    - 그 외 React 에러, import 에러, TypeError 등은 모두 보고합니다.
69. **노란색 경고**가 있는가? React key 경고, deprecated API 경고 등이 있으면 기록합니다.

### 주의사항

1. **백엔드가 없습니다.** 1~6번 앱(Terminal, VS Code, Claude, Monitor, Kasm, Settings)은 백엔드 API를 호출하므로 정상 동작하지 않을 수 있습니다. 이 앱들의 "연결 실패/API 키 미설정/로딩 중" 메시지는 **정상**이므로 실패로 기록하지 마세요.
2. 7~12번 앱(Profile, Resume, Projects, Blog, Timeline, Contact)은 프론트엔드만으로 동작하는 **순수 클라이언트 앱**입니다. 이 앱들에서 발생하는 문제가 진짜 버그입니다.
3. 스크린샷을 찍을 수 있다면 실패 항목마다 첨부해주세요.
4. 같은 문제가 여러 곳에서 반복되면 한 번만 상세히 기록하고 나머지는 "항목 N과 동일" 처리하세요.
5. 윈도우를 **매우 작게 리사이즈**했을 때 레이아웃이 깨지는 것도 기록해주세요.
