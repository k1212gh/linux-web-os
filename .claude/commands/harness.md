# Phase별 자동 실행 하네스

주어진 작업을 아래 Phase 순서로 체계적으로 실행합니다.

## Phase 순서

### Phase 1: 컨텍스트 수집
- `.claude/rules/` 디렉토리의 관련 규칙 파일 읽기
- 관련 파일/디렉토리 구조 파악
- 기존 코드 패턴 분석

### Phase 2: 계획
- 변경할 파일 목록 작성
- 구현 순서 결정
- 사용자에게 계획 보고 (승인 대기)

### Phase 3: 구현
- 계획에 따라 코드 작성
- 각 파일 수정 후 즉시 빌드 검증
- 에러 발생 시 수정 → 재검증

### Phase 4: 검증
- `cd frontend && ./node_modules/.bin/vite build` 실행
- 빌드 에러 0 확인
- 변경 사항 요약

### Phase 5: 커밋
- `git add` (변경 파일만, -A 금지)
- Conventional Commit 메시지 작성
- 사용자 승인 후 커밋

## 서킷 브레이커
- 같은 에러 3회 반복 시 **중단하고 사용자에게 보고**
- 빌드 실패 3회 시 중단
- 확실하지 않은 변경은 하지 않음

## 규칙
- `.env*` 파일 절대 읽지 않음
- 사용자가 명시하지 않은 파일 수정 금지
- 기존 코드 스타일 유지 (인라인 스타일, Zustand 패턴)
