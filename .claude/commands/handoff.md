# 기기 이동 전 상태 저장 (/handoff)

현재 세션의 작업 내용을 정리하여 다른 기기에서 이어받을 수 있게 합니다.

## 절차

1. **현재 작업 상태 파악**
   - `git status`로 변경된 파일 확인
   - `git log --oneline -5`로 최근 커밋 확인
   - 열려있는 이슈나 진행 중인 작업 파악

2. **SESSION_CONTEXT.md 업데이트**
   - "마지막 작업" 섹션에 오늘 한 일 기록
   - "다음 할 일" 섹션에 이어서 해야 할 작업 기록
   - 발견된 버그나 이슈 기록
   - 현재 기기 정보 (어떤 기기에서 작업했는지)

3. **커밋되지 않은 변경 사항 커밋**
   - 변경 파일을 확인하고 적절한 커밋 메시지로 커밋
   - 작업 중이던 미완성 코드도 `wip:` prefix로 커밋

4. **푸시**
   - `git push origin feat/portfolio-apps`

5. **확인 보고**
   ```
   ✅ Handoff 완료
   - 커밋: [해시] [메시지]
   - 다음 기기에서: git pull → /resume
   - 주의 사항: [있으면 기록]
   ```

## SESSION_CONTEXT.md 업데이트 형식

파일 끝에 추가:
```markdown
---
## 마지막 작업 (YYYY-MM-DD, 기기명)
- 작업 1
- 작업 2

## 다음 할 일
1. 첫 번째 할 일
2. 두 번째 할 일

## 이슈/주의
- 이슈 내용
```
